## 01-1 컴퓨터 구조를 알아야 하는 이유

> 컴퓨터 구조를 이해하면 **문제 해결 능력**이 향상된다.
> 컴퓨터 구조를 이해하면 문법만으로는 알기 어려운 **성능/용량/비용**을 고려하며 개발할 수 있다.


### 문제 해결

- 문제 상황 빠르게 진단 가능
- 문제 해결 실마리 다양하게 탐색 가능

### 성능, 용량, 비용

- 문법만으로는 알기 어려운 부분을 고려하며 개발 가능
(ex. 서버 컴퓨터 or 클라우드 서비스 이용 시 CPU, 메모리, 저장 용량 선택)

## 01-2 컴퓨터 구조의 큰 그림

> **데이터와 명령어** : 컴퓨터가 이해하는 정보
**메모리** : 현재 실행되는 프로그램의 **명령어/데이터 저장**하는 부품
**CPU** : 메모리에 저장된 명령어를 **읽어 들이고, 해석하고, 실행**하는 부품
**보조기억장치** : **전원이 꺼져도** 보관할 프로그램을 저장하는 부품
**입출력장치** : 컴퓨터 외부에 연결되어 컴퓨터 내부와 정보 교환하는 부품
**시스템 버스** : 컴퓨터의 네 가지 핵심 부품들이 서로 정보를 주고받는 통로
> 

### 컴퓨터가 이해하는 정보

- 데이터 : 정적인 정보 (ex. 숫자, 문자, 이미지, 동영상)
- 명령어 : 데이터를 움직이고, 컴퓨터를 작동시키는 정보

### 컴퓨터의 4가지 핵심 부품

![computer-structure](/start-computer-structure/computer-structure.png)

- **중앙처리장치(CPU : Central Processing Unit)** : 메모리에 저장된 값을 읽어 들이고, 해석하고, 실행하는 장치
    - **산술논리연산장치(ALU: Arithmetic Logic Unit)** : 컴퓨터 내부 수행 계산
    - **레지스터(register)** : 프로그램 실행 시 필요 값 임시 저장 공간
    - **제어장치(CU: Control Unit)** : 제어 신호 전송 및 명령어 해석 장치
        - 메모리 읽기 : CPU가 메모리에 저장된 값을 읽고 싶을 때 메모리를 향해 보내는 제어 신호
        - 메모리 쓰기 : CPU가 메모리에 어떤 값을 저장하고 싶을 때 메모리를 향해 보내는 제어 신호
- **주기억장치(Memory : main memory)** : 현재 실행되는 프로그램의 명령어와 데이터를 저장하는 부품
    - 프로그램이 실행되려면 반드시 메모리 저장 필요
    - 메모리에 저장된 값의 위치는 주소로 파악 가능
    - 종류
        - RAM(Random Access Memory) : 사용자의 실제 작업 공간, 보통의 주기억장치
        - ROM(Read Only Memory) : 컴퓨터 부팅 시 필요한 기본적 명령어, 펌웨어 (일반적으로 사용자가 데이터를 쓰고 지울 수 없음)
- **보조기억장치(secondary storage)** : 전원이 꺼져도 저장된 내용을 잃지 않는 메모리를 보조할 저장 장치
    - 하드 디스크, SSD, USB 메모리, DVD, CD-ROM
- **입출력장치(input/output(I/O) device)** : 컴퓨터 외부에 연결되어 컴퓨터 내부와 정보를 교환하는 장치
    - 마이크, 스피커, 프린터, 마우스, 키보드

### 메인보드와 버스

- 메인 보드(main board) : 마더 보드
- 버스 : 메인보드에 연결된 부품들이 서로 정보를 주고받을 수 있도록 하는 통로
    - 시스템 버스 : 컴퓨터 네 가지 핵심 부품의 연결 버스
        - 주소 버스 : 주소 주고받는 통로
        - 데이터 버스 : 명령어/데이터 주고받는 통로
        - 제어 버스 : 제어 신호 주고받는 통로
